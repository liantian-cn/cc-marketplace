# Bing 搜索 — 参数参考

Bing 搜索提供两个独立引擎：**Bing CN**（必应中国，cn.bing.com）和 **Bing Int**（必应国际，www.bing.com）。两者均**无需 API Key**，通过网页抓取获取搜索结果。

## 两个引擎的定位

| 引擎 | 站点 | 代理 | 最佳场景 |
|------|------|------|----------|
| **Bing CN** | cn.bing.com | 不需要 | 中文+英文混合查询，中英双语结果，国内可直接访问 |
| **Bing Int** | www.bing.com | **需要** (中国大陆) | 英文查询，国际视野，英文结果更丰富 |

## 调用方式

Bing 搜索没有 MCP 服务，通过 **直接 Bash 命令** 执行本地 Python 脚本：

```bash
# Bing CN（必应中国）
python plugins/essentials/skills/advanced-search/scripts/bing_cn_no_api.py "关键词" -n 数量

# Bing Int（必应国际，需代理）
python plugins/essentials/skills/advanced-search/scripts/bing_int_no_api.py "关键词" -n 数量 --proxy http://127.0.0.1:7890
```

**关键区别：** Bing 与 WeChat 一样 — 不使用 MCP 工具，不需要子代理。在并行执行阶段直接通过 Bash 调用脚本。

## 参数

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `query` | string | (必填) | 搜索关键词。Bing CN 支持中英文混合查询，Bing Int 偏向英文查询 |
| `-n, --num` | integer | 10 | 返回结果数量，范围 1–50 |
| `-o, --output` | string | (可选) | 输出 JSON 文件路径 |
| `--proxy` | string | `$BING_PROXY` | **仅 Bing Int**。代理地址，也可通过 `BING_PROXY` 环境变量设置 |

## 响应结构

```json
{
  "query": "搜索关键词",
  "total": 10,
  "results": [
    {
      "title": "结果标题",
      "url": "https://...",
      "summary": "页面摘要/片段"
    }
  ]
}
```

**错误响应：**
```json
{
  "query": "关键词",
  "total": 0,
  "results": [],
  "error": "错误描述"
}
```

## Bing CN vs Bing Int 选择指南

| 场景 | 推荐 |
|------|------|
| 中文查询或中英混合查询 | **Bing CN** — 直连速度快，中英双语结果 |
| 纯英文查询，需要国际视野 | **Bing Int** — 全球英文结果更丰富 |
| 搜索国外技术文档 | **Bing Int** — 英文技术内容覆盖面更广 |
| 搜索国内新闻/话题 | **Bing CN** — 中文结果更相关 |
| 没有代理可用 | **仅 Bing CN** — cn.bing.com 国内直连 |

## 局限与风险

- **网页抓取不稳定** — 页面结构可能变化导致解析失效，这是所有无 API 抓取工具的固有限制
- **无时间过滤** — 无法按时间范围筛选，只能通过关键词中包含时间词来间接影响
- **无全文提取** — 仅返回标题和摘要，不提供正文抓取
- **Bing Int 需代理** — 中国大陆无法直接访问 www.bing.com，必须配置代理
- **Bing CN 结果有限** — cn.bing.com 的结果数量和质量不如国际版丰富
- **反爬虫风险** — 过于频繁的请求可能触发验证码或 IP 封禁
